## Odoo simbalirion Custom Sale Module v5

**Модуль для Odoo Community Edition v18**

Добавляет обязательное поле "Ответственный за выдачу товара" и текстовое поле New Field в форму коммерческого предложения / заказа на продажу.

## Описание функционала

Модуль расширяет стандартный модуль Sale и добавляет:

- **Поле "Ответственный за выдачу товара"**

  - Тип: Many2one (выбор сотрудника из Employee)

  - Обязательное для заполнения в статусе Draft.

- **Поле "New Field"**

  - Тип: Char

  - При создании нового Quotation автоматически заполняется случайным набором 10 букв.

  - При изменении дат заказа или строк заказа (order_line) поле автоматически обновляется в формате: DD/MM/YYYY HH:MM:SS + сумма

  - Если пользователь ввел значение вручную (не сгенерированное системой), оно не перезаписывается.

  - Максимальная длина текста: 30 символов. При превышении — появляется ошибка валидации.

  - Доступность поля по статусам:

    - Draft — редактируемое

    - Sent — только для чтения

    - Sale — скрыто

  - Отображение в печатной форме

    - Если поле заполнено, отображается над таблицей товаров:

      - New Field: <значение>

    - Если поле пустое, выводится сообщение:

      - Поле New Field пустое!


## Установка

1. Скопируйте папку модуля odoo_simbalirion_v4 в директорию addons вашего инстанса Odoo.

2. Перезапустите Odoo сервер:

```
./odoo-bin -c <your-config-file>.conf
```

3. В интерфейсе Odoo откройте Apps → Update Apps List.

4. Найдите модуль Odoo simbalirion Custom Sale v4 и установите.

## Использование

1. Перейдите в Sales → Orders → Create для создания нового Quotation.

2. Убедитесь, что:

- Поле Ответственный за выдачу товара отображается и обязательно для заполнения.

- Поле New Field автоматически заполнено случайным набором букв.

- Измените дату заказа или добавьте/удалите товары:

- Если поле не редактировалось вручную, оно обновится на формат Дата + Сумма.

- Если поле вводилось вручную, значение не изменяется.

- Подтвердите Quotation, чтобы увидеть соответствующий статус (Sent, Sale) и ограничения редактирования/видимости поля.

- Для просмотра поля в печатной форме используйте Preview или Print Quotation/Order.

## Техническая реализация

Модель расширяет sale.order через _inherit.

**Поля:**

```
responsible_employee_id = fields.Many2one('hr.employee', required=True)

new_field = fields.Char(default=lambda self: ''.join(random.choices(string.ascii_letters, k=10)))
```

**Валидация длины:** 

```
@api.constrains('new_field')
```

**Автообновление при изменении строки или даты:** 

```
@api.onchange('order_line', 'date_order')
```

**QWeb-шаблон для печатной формы:**

- Использует t-foreach="docs" и t-if="o.new_field" для корректного отображения значения или сообщения о пустом поле.

**Поддержка статусов:**

- Draft — редактируемое

- Sent — только чтение

- Sale — скрыто через invisible="state == 'sale'" и readonly="state == 'sent'"

**Зависимости**

- sale

- hr

**Примеры**

- New Field автоматически заполнено при создании Quotation

```
qwertyuiop
```

- New Field после изменения даты/товаров (если не редактировалось вручную)

```
19/03/2026 18:43:04 + 8287.50
```

- Пустое поле в печатной форме

```
Поле New Field пустое!
```
