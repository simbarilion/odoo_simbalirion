## Odoo simbalirion Custom Sale

**Модуль для Odoo Community Edition v18**

Расширяет стандартный модуль **Sale** — добавляет обязательное поле «Ответственный за выдачу товара» 
и специальное текстовое поле **New Field** с автоматической генерацией и обновлением.

## Функционал

**Поле "Ответственный за выдачу товара"**

- Тип: Many2one (выбор из сотрудников — hr.employee)  

- Обязательное (required=True)  

- Появляется в форме Quotations / Sales Orders сразу после поля партнёра

**Поле "New Field"**

- Тип: Char

- При создании нового Quotation автоматически заполняется случайным набором 10 букв.

- При изменении **даты заказа** (`date_order`) или **строк заказа** (`order_line`) — **автоматически обновляется** в формате:  
  `DD/MM/YYYY HH:MM:SS + сумма_с_двумя_знаками`

  Пример: `20/03/2026 14:30:00 + 1234.56`

- Если пользователь **вручную отредактировал** поле (хоть на один символ), автоматическое обновление отключается навсегда (реализовано через флаг `new_field_auto_generated`)  

- Ограничение длины: **не более 30 символов** (серверная валидация с понятным сообщением)  

- **Поведение в зависимости от статуса документа**  
  - **Draft** — поле полностью редактируемо  
  - **Sent** — поле только для чтения (readonly)  
  - **Sale** — поле полностью скрыто (invisible)

## Установка

1. Скопируйте папку модуля odoo_simbalirion_v10 в директорию addons вашего инстанса Odoo.

2. Создайте в корне инстанса Odoo файл odoo.conf и заполните его значениями:

```aiignore
Пример:

[options]
addons_path = addons,custom_addons
db_host = localhost
db_port = 5432
db_user = odoo
db_password =
```

2. Перезапустите Odoo сервер:

```bash
python odoo-bin -c odoo.conf -u all
```

3. В интерфейсе Odoo откройте Apps → Update Apps List.

4. Найдите модуль Odoo simbalirion Custom Sale v10 → Install.

## Использование

1. Перейдите в Sales → Orders → Create для создания нового Quotation.

2. Проверьте:

- Поле "Ответственный за выдачу товара" отображается и обязательно для заполнения.

- Поле "New Field" автоматически заполнено случайными 10 буквами.

3. Измените дату заказа или добавьте/удалите товары:

- Если поле не редактировалось вручную, оно обновится на формат "Дата + Сумма".

- Если поле вводилось вручную, значение не изменяется.

4. Попробуйте ввести текст длиннее 30 символов → при сохранении увидите ошибку.

5. Подтвердите КП (Send by Email / Confirm Sale), чтобы увидеть соответствующий статус (Sent, Sale) 
и ограничения редактирования/видимости поля.

## Техническая реализация

Наследование модели: _inherit = "sale.order"

**Поля:**

```
responsible_employee_id = fields.Many2one("hr.employee", required=True)

new_field = fields.Char(default=lambda self: "".join(random.choices(string.ascii_letters, k=10)))

new_field_auto_generated = fields.Boolean(default=True)  # флаг авто-генерации
```

**Валидация длины:** 

```
@api.constrains("new_field")
```

**Автообновление при изменении строки или даты:** 

```
@api.onchange("order_line", "date_order")
```

**Отключение авто-режима при ручном редактировании:**

```
@api.onchange('new_field')
```

**Поддержка статусов:**

- Draft — редактируемое

- Sent — только чтение

- Sale — скрыто через invisible="state == 'sale'" и readonly="state == 'sent'"

**Зависимости**

- sale

- hr

**Примеры**

- New Field автоматически заполнено при создании Quotation

```
qwertyuiop
```

- New Field после изменения даты/товаров (если не редактировалось вручную)

```
19/03/2026 18:43:04 + 8287.50
```

**Примечание:** отображение поля в печатной форме (Quotation / Order PDF) не реализовано из-за сложностей 
с наследованием QWeb-шаблона в текущей установке (Studio / переопределения шаблона).
